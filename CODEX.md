# Codex — Project Instructions

## Общие правила для ИИ

1. **Всегда отвечай на русском языке**, если пользователь явно не попросил другой язык.
2. **Проверяй актуальность версий через интернет** перед утверждениями о последних релизах (сейчас апрель 2026).
3. **Не предлагай косметические правки**, если код уже соответствует конвенциям проекта.
4. **Не повторяй уже обсуждённые проблемы** в рамках текущего диалога.
5. **Давай конкретику:** команда, патч, файл, строка, а не общие рекомендации.
6. **После каждого изменения файла делай `git add` + `git commit` + `git push`.**
7. **После завершения задачи запускай `go build ./...` и `go vet ./...`.**

## О проекте

- **Название:** websocket-chat
- **Язык:** Go 1.26
- **Технологии:** WebSocket (gorilla/websocket), JWT, PostgreSQL (pgxpool), embed, Docker
- **Структура:**
  - `cmd/server/` — точка входа
  - `internal/auth/` — аутентификация
  - `internal/hub/` — WebSocket Hub
  - `internal/store/postgres/` — хранение данных
  - `internal/transport/` — HTTP/WebSocket хендлеры
  - `internal/domain/` — доменные модели
  - `pkg/protocol/` — протокол/обёртка над websocket
  - `web/` — фронтенд
  - `migrations/` — SQL-миграции

## Конвенции

- **Go-нейминг:** camelCase / PascalCase для экспортируемых
- **SQL:** snake_case (`user_id`, `created_at`)
- **JSON:** camelCase
- **Горутины:** суффиксы `Pump`/`Goroutine`
- **Ошибки:** `err` или `varNameErr`
- **Комментарии:** документальные для экспортируемых сущностей
- **Внутри функций:** комментарии только при реальной необходимости

## Базовые команды

- `go build ./...`
- `go test ./...`
- `go vet ./...`
- `golangci-lint run ./...`
- `docker-compose up`

## Review и Git

1. При запросе ревью сначала проверяй потенциальные баги/регрессии и отсутствие тестов.
2. Критичные проблемы предлагай исправлять сразу.
3. Работай в feature-ветках; `main` — только через PR после ревью.
4. Формат коммитов:
   - `feat: ...`
   - `fix: ...`
   - `docs: ...`
   - `refactor: ...`
